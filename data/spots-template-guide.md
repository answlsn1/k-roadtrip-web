# K-Riders 스팟 CSV 템플릿 가이드

`data/spots-template.csv` 작성 규칙. 필드 사전의 단일 기준은
`lib/motorcycle/spots/types.ts` + `constants.ts` — 이 문서는 그 기준을 사람이 읽기 좋게 옮긴 것이다.

## 사용 순서

1. 시트(엑셀/구글시트)에서 스팟 정리 — 이 템플릿의 컬럼(28개)을 그대로 사용
2. CSV(UTF-8)로 내보내기 — 엑셀은 "CSV UTF-8(쉼표로 분리)" 선택
3. `npm run spots:validate -- 파일경로` 로 검증 (인자 없으면 `data/spots-template.csv` 대상)
4. 전부 통과하면 6단계 import 로 DB(`motorcycle_spots`) 반영

## 공통 주의사항

- **템플릿의 예시 3행(언덕끝 로스터리 / 달빛재 전망쉼터 / 은행나무재 남측 진입점)은 가상 데이터다 — 반드시 지우고 사용할 것.**
- **리스트 값은 세미콜론(`;`)으로 구분**한다 (`tags`/`photos`/`best_season`/`best_time`).
  쉼표는 CSV 자체 구분자라 충돌한다 — 절대 쉼표로 리스트를 만들지 말 것.
- 파일 인코딩은 UTF-8. 템플릿은 엑셀 한글 깨짐 방지용 BOM 이 이미 포함돼 있다.
- 셀 값에 쉼표(,)를 쓰려면 시트에서 내보낼 때 자동으로 따옴표 처리되는지 확인할 것(가급적 쉼표 미사용 권장).
- 좌표는 십진수 도(deg) 단위, 소수점 4자리 이상 권장 (예: `35.9873`). 한국 범위: 위도 33~39 / 경도 124~132 — 벗어나면 오타로 간주해 실패한다.
- 중복 판정: **같은 이름(트림 후 완전일치) 또는 100m 이내 좌표**는 파일 내/기존 DB 와 중복으로 걸린다.
- `slug` 를 비워두면 `name` + `region_sido` 로 자동 생성된다 (예: "카페 언덕집" + "경북" → `카페-언덕집-경북`).
- 속도/랩타임/랭킹 관련 값은 어떤 필드에도 적지 않는다 (`winding_grade` 는 재미 등급이지 속도 지표가 아님).

## 컬럼 사전

| 컬럼 | 필수 | 타입/허용값 | 작성 예시 | 주의점 |
|---|---|---|---|---|
| `name` | **필수** | 텍스트 | `언덕끝 로스터리` | 간판/통칭 그대로. 파일 내·기존 DB 와 이름이 같으면 중복 실패 |
| `category` | **필수** | `카페` `전망포토` `와인딩진입점` `맛집` `휴게쉼터` `주유충전` `정비` `숙박` `기타` | `카페` | 목록 밖 값은 실패 |
| `lat` | **필수** | 숫자(위도) | `35.9873` | 33~39 범위 밖이면 실패 |
| `lng` | **필수** | 숫자(경도) | `128.6841` | 124~132 범위 밖이면 실패 |
| `region_sido` | **필수** | 텍스트(시/도) | `대구`, `경북` | slug 자동 생성에 쓰인다 |
| `source` | **필수** | `직접방문` `크루추천` `인스타` `네이버` `유튜브` `기타` | `직접방문` | 목록 밖 값은 실패 |
| `slug` | 선택 | 소문자 영문/숫자/한글 + 하이픈 | `카페-언덕집-경북` | **비우면 자동 생성**. 채우면 형식 검사 + 충돌 검사 |
| `alt_name` | 선택 | 텍스트 | `언덕집` | 별칭/구 상호 |
| `tags` | 선택 | 텍스트 리스트 | `뷰맛집;테라스` | **세미콜론(;) 구분** |
| `address_road` | 선택 | 텍스트 | `대구 동구 팔공산로 1234` | 도로명 주소 |
| `region_sigungu` | 선택 | 텍스트(시/군/구) | `동구`, `군위군` | |
| `access_note` | 선택 | 텍스트 | `오르막 진입로 폭 좁음` | 진입로/접근 팁 |
| `road_surface` | 선택 | `포장` `일부비포장` `비포장` | `포장` | 목록 밖 값은 실패 |
| `parking_moto` | 선택 | `양호` `보통` `협소` `불가` | `양호` | 모터사이클 주차 기준 |
| `winding_grade` | 선택 | 정수 1~5 | `4` | 인근 와인딩 재미 등급 — **속도 지표 아님** |
| `senior_friendly` | 선택 | `true`/`false`/`1`/`0`/`예`/`아니오` | `true` | 초보/시니어 라이더 부담 없는 곳인지 |
| `photo_spot` | 선택 | `true`/`false`/`1`/`0`/`예`/`아니오` | `true` | |
| `photo_note` | 선택 | 텍스트 | `일몰 30분 전 도착 추천` | 촬영 포인트 메모 |
| `best_season` | 선택 | 월 숫자 리스트(1~12) | `4;5;10` | **세미콜론(;) 구분** |
| `best_time` | 선택 | `일출` `오전` `오후` `일몰` `야간` 리스트 | `오후;일몰` | **세미콜론(;) 구분**, 목록 밖 값은 실패 |
| `stay_minutes` | 선택 | 정수(분, 0 초과) | `40` | 권장 체류 시간 |
| `crowd_weekend` | 선택 | `한적` `보통` `붐빔` | `한적` | 주말 기준 혼잡도 |
| `rating_personal` | 선택 | 정수 1~5 | `4` | 운영자 개인 평점 |
| `verified` | 선택 | `true`/`false`/`1`/`0`/`예`/`아니오` | `false` | 실방문 검증 여부. 비우면 DB 기본 `false` |
| `source_url` | 선택 | URL 텍스트 | | 출처 링크(인스타/유튜브 등) |
| `memo` | 선택 | 텍스트 | `낙엽 시즌 노면 주의` | 자유 메모 |
| `photos` | 선택 | Storage 경로 리스트 | | **세미콜론(;) 구분**. 보통 비워두고 6단계 이후 업로드 |
| `status` | 선택 | `draft` `active` `hidden` | `draft` | 비우면 DB 기본 `draft`. `active` 만 공개 노출 |

## 검증 실패가 나면

`npm run spots:validate` 출력의 행 번호는 **사람 기준**(헤더=1행, 데이터는 2행부터) —
시트에서 같은 행 번호를 바로 찾아 고치면 된다. 실패가 1건이라도 있으면 exit code 1 로 끝난다(6단계 import 진행 금지).
