# CLAUDE.md — K-RoadTrip 런칭 헌장 (Launch Harness)

> 이 파일은 K-RoadTrip 저장소의 모든 Claude Code 세션이 자동으로 읽는다. 이 앱의 목표는 **"확실한 런칭"**이며, 일반 작업보다 엄격한 규율을 적용한다. 받은 개발지시도, 모든 검수도 이 헌장을 기준으로 한다.

## 1. 이 앱이 무엇인가 (절대 잊지 말 것) — v2
K-RoadTrip은 **외국인을 위한 한국 지방 로드트립 발견·기획·예약 레이어**다. 지도/내비 *위*에 얹혀, 큐레이션된 지방 루트를 **발견**하고, 일정을 **기획**하고, 렌터카·항공을 **예약**(제휴)하게 한다.
- 핵심 가치 = 큐레이션·발견·예약. **메시지: "서울 너머, 완벽한 한국 로드트립을 계획·예약."**
- 내비는 **commodity** — 가치의 중심이 아니다. 어떤 지도로든 **원탭 핸드오프**(네이버 지금 + 구글 출시 시 옵션 + 사용자 선택). *2026.2 구글 지도 내비 반출 허가로 내비 갭은 사라지는 중 → 내비에 포지션을 기대지 않는다.*
- 운영: **1인 운영(solo).** 손 많이 가는 구조 금지.
- 제품 우선순위(v2): 일정 큐레이션 · 지방 발견 · 예약 funnel · 외국인 컨텍스트 · 플랜 저장/공유/수정. 지도는 시각화(OSM/Leaflet)만 — **지도 자체가 되려 하지 말 것.**

## 2. 수익은 어떻게 나는가 (= 가장 보호해야 할 것)
수익 = **제휴 전환**. 트래픽 → **발견 → 기획(여행 계획) → 예약(렌터카·항공 제휴 링크 클릭)** → 결제 → **수수료**. **"계획 → 예약"이 이 앱의 핵심 가치 흐름이다.**
즉 **제휴 링크와 그 추적(tracking)이 곧 매출이다.** 링크가 깨지거나, 추적 파라미터가 빠지거나, 전환이 귀속되지 않으면 **매출은 0이다.** 이 경로는 신성하게 다룬다.
부가 가치: 외화 유입 + 지방도시 여행 활성화 → 지자체 협력 가능성.

## 3. 기술 스택 (고정)
Next.js (App Router) + TypeScript(strict) / Supabase(Postgres + RLS) / Vercel.

## 4. "런칭 가능(Launch-Ready)"의 정의
아래가 **전부** 충족되기 전엔 런칭 가능이 아니다 (`LAUNCH-READINESS.md` 참조). **런칭은 binary다** — 핵심 항목 하나라도 미충족이면 런칭 불가.
1. 모든 루트·장소·미디어가 **실제·현지검증** 데이터 (더미 0)
2. **제휴 전환 funnel 동작** — 링크 실재·추적·결제 경로 검증
3. **지도 핸드오프 무결** — 좌표 정확·동작·실패 폴백 (네이버 지금, 구글 출시 시 옵션, 사용자 선택). *기능이지 존재 이유는 아님.*
4. **i18n 완성** — 영어 100%(한국어 누출 없음), 한국어 토글 동작
5. 해외 접속 **성능**·모바일 네이티브
6. **신뢰·일관성** — 메시지 모순 없음, 제휴 disclosure 존재
7. 외부 API 실패 **graceful 처리**

## 5. 절대 규칙 (Never — 런칭 하드룰)
- 프로덕션 경로에 **플레이스홀더·더미·외부 샘플 미디어 금지** (Big Buck Bunny, MDN cc0-videos, learningcontainer 등 샘플 URL 절대 금지)
- **제휴/결제/시크릿/프로덕션 DB**를 건드리는 변경은 **사람 승인 없이 진행 금지** ← 가드레일
- 시크릿·제휴 키를 클라이언트 번들이나 `NEXT_PUBLIC_*`에 노출 금지
- 영어 화면에 한국어가 새거나, 미번역 문자열을 남긴 채 배포 금지
- 외부 API(지도·제휴·미디어) 실패 처리를 안 한 채 배포 금지
- "광고 없음" 류 주장과 스폰서 콘텐츠를 **모순되게** 방치 금지
- Supabase RLS 항상 ON

## 6. 받은 개발지시 처리 규율 (Dev-Instruction Harness)
지시를 받으면:
1. **범위·영향 복창** 후 시작 (무엇을, 어디를, 무엇이 영향받는지)
2. 지시가 **런칭 크리티컬 영역**(제휴·결제·지도 핸드오프·i18n·시크릿·프로덕션 DB)을 건드리면 → **먼저 경고하고 사람 승인을 받는다.** 독단 변경 금지.
3. 편의를 위해서라도 **더미·플레이스홀더를 프로덕션 경로에 도입하지 않는다** (개발용은 명확히 분리·표시).
4. 스펙 갭·모순은 추측으로 메우지 말고 **에스컬레이션**.
5. 모든 변경은 **`launch-reviewer` 통과 후에만** "완료"로 보고.
