# K-Riders — REVER 벤치마크 & 로드맵

> 목표: 해외 1위 모터사이클 커뮤니티 앱 **REVER**(rever.co)를 벤치마킹해
> K-Riders(/motorcycle)를 "대한민국의 REVER" 수준으로 끌어올린다.
> 원칙: 무료 티어(Supabase/Vercel)로 가능한 것 먼저, 유료·네이티브 앱이
> 필요한 것은 틀만 잡고 로드맵에 명시(가짜 기능 금지 — CLAUDE.md §5).

## 1. REVER 기능 전수 분석 (2026-07-02 조사)

| REVER 기능 | 설명 | K-Riders 대응 |
|---|---|---|
| 라이드 기록(GPS 트래킹) | 주행 경로·거리·시간 기록, 공개/비공개 | ✅ **P1 구현** — 웹 Geolocation 기반 기록 모드 |
| 루트 플래닝 | 지도에서 경유지 찍어 루트 생성 | ✅ 이미 구현(/routes/new) |
| 커뮤니티 피드 | 공개 라이드 발견, 좋아요/댓글 | ✅ **P1 구현** — 좋아요·댓글·정렬·필터 |
| 라이더 팔로우 | 라이더 간 팔로우, 프로필 | ✅ **P1 구현** — 팔로우 + 공개 프로필 페이지 |
| 개인 통계 | 누적 거리·라이드 수 | ✅ **P1 구현** — 프로필 통계 |
| 챌린지 | 기간/거리 챌린지, 뱃지 | ✅ **P1 구현(경량)** — 거리 마일스톤 배지 + 이달의 챌린지 |

| 날씨 레이더 | 유료 API | ⏳ 유료 API 필요 — 틀만(로드맵) |

| 사진 업로드 | 라이드에 사진 첨부 | ⏳ **P2** — Supabase Storage(무료 1GB) 버킷 셋업 별도 필요 |

## 2. 오늘(P1) 구현 범위

**소셜 레이어** — 마이그레이션 `0007_motorcycle_social.sql`
- 좋아요(`motorcycle_route_likes`), 댓글(`motorcycle_route_comments`), 팔로우(`motorcycle_follows`)
- 프로필 확장: `bio` 컬럼
- 루트 확장: `track_points`(기록 주행 GPS 폴리라인, jsonb), `duration_min`

**주행 기록 모드** (/motorcycle/record) — REVER의 코어 기능
- 브라우저 Geolocation `watchPosition` 기반, 실시간 거리/시간 표시
- 종료 시 제목/지역/공개여부 입력 → 루트로 저장(트랙 폴리라인 포함)
- ⚠️ 웹 한계 honest 고지: 화면이 꺼지면 기록이 끊길 수 있음(백그라운드
  GPS는 네이티브 앱 영역 — P3)

**라이더 프로필** (/motorcycle/riders/[id])
- 닉네임·바이크·소개, 누적 통계(루트 수·총 거리), 배지, 팔로우 버튼, 공개 루트 그리드

**배지 & 챌린지(경량)** — `lib/motorcycle/badges.ts`
- 누적 거리 마일스톤(첫 시동/100km/300km/633km 국토종주급/1,500km/3,000km)
- "이달의 챌린지": 이번 달 누적 거리 프로그레스(클라이언트 계산, 테이블 0)

**내비게이션** — 루트 상세에 "네이버 지도로 길안내"(`buildNaverWebRouteUrl` 재사용)

## 3. P2 (다음 스프린트, 무료 가능)
- 라이드 사진 업로드(Supabase Storage 버킷 + 정책 + 업로드 UI)
- 댓글 알림(인앱), 팔로잉 피드 탭(팔로우한 라이더 루트만)
- 운영자 큐레이션 루트(국내 와인딩 명소 — Butler 지도의 한국판)

## 4. P3 (유료/네이티브 앱 필요 — 틀만, 절대 가짜 UI 금지)
- 실시간 친구 위치 공유(LiveRIDE류) — 백그라운드 GPS = 앱
- 날씨 레이더 오버레이 — 유료 기상 API
- 오프라인 지도 / 자체 턴바이턴 내비 — 앱
- K-Riders Pro(구독) — 위 기능들 묶음, 결제(Stripe) 연동 시점에 설계
