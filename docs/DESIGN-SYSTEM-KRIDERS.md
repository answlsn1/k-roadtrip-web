## 2. 토큰

- **서피스 3단**: `--kr-bg #0c0e12`(기저) → `--kr-surface-1 #14171c`(카드) →
  `--kr-surface-2 #1b2027`(호버/입력). 경계선 `--kr-line`(8% 백색)/`--kr-line-strong`(16%).
  기저 배경에 상단 앰버 라디얼 글로우(7%/4%) — 브랜드 온도를 배경에서 암시.
- **액센트**: amber-500 단일 — 버튼 배경·활성 상태·데이터 강조에만. 텍스트 링크
  호버는 amber-400/300.
- **타이포**: 페이지 제목 `text-2xl sm:text-3xl font-extrabold tracking-tight text-white`,
  킥커 `text-[11px] font-bold uppercase tracking-[0.18em] text-amber-500`,
  본문 뮤트 `text-slate-400`, 메타 `text-xs text-slate-500`.
- **라운드**: 카드 20px(rounded-[1.25rem]), 입력 14px, 버튼 풀 라운드.
- **모션**: 150ms(버튼)·200ms(카드)·350ms(등장). `prefers-reduced-motion` 전면 존중.

## 3. 컴포넌트 규칙 (CSS 클래스 → 사용처)

- `.kr-btn-primary` — 앰버 채움+앰버 섀도. 화면당 1~2개(주행동선)만.
- `.kr-btn-secondary` — 보더 필. 보조 액션(공유·GPX·팔로우 해제 상태).
- `.kr-btn-danger` — 삭제류.
- `.kr-card` (+`.kr-card-hover`) — 서피스1, 호버 시 서피스2+보더 앰버+lift 2px+딥 섀도.
- `.kr-input` — 서피스2 + 포커스 앰버 보더+링.
- `.krider-map-dark` — 지도 래퍼에 부여: OSM 타일 invert 다크 변환 + 컨트롤/어트리뷰션 다크.
- `.kr-skeleton` — shimmer 로딩.
- `.kr-heart` — 좋아요 SVG 에 부여 시 aria-pressed 전환에서 팝 애니메이션(순수 CSS).
- 포커스 링: 스코프 전역 자동(앰버 2px outline).

## 4. 화면별 지침 요약

- **헤더**: usePathname 기반 활성 나브(앰버 텍스트+하단 2px 앰버 바), 배경
  `bg-[#0c0e12]/85` + blur, 경계 `--kr-line`.
- **피드 히어로**: 좌측 정렬 전환, 수직 패딩 축소(py-10~14), 킥커+타이틀+서브만.
  글로우는 scope 배경이 담당 — 히어로 자체 장식 최소.
- **RouteCard**: kr-card+hover, 거리·시간을 흰색 강조, 아이콘 SVG 통일
  (하트=기존 SVG, 댓글=말풍선 SVG — 이모지 금지).
- **지도 3종**(상세/기록/작성): 래퍼 div 에 krider-map-dark. 앰버 폴리라인이
  다크 타일 위에서 주인공이 됨.
- **빈 상태 공통 구성**: 56px 원형(서피스2) 안 아이콘 → 제목(흰 볼드) → 서브(뮤트)
  → CTA(프라이머리). 가운데 정렬, py-14.
- **폼(글쓰기·회원가입·로그인·프로필 수정·기록 저장)**: 전 입력 kr-input,
  제출 kr-btn-primary, 레이블 text-sm font-semibold text-slate-300 유지.
- **채팅**: 내 버블 앰버(rounded-2xl, tr 만 sm), 상대 서피스2. 입력바는 하단
  고정 카드(border-t --kr-line, 배경 --kr-bg/90+blur).
- **게시판 목록**: 행 호버 bg-white/[0.04], 제목 호버 시 앰버 전환 150ms,
  댓글 수 앰버 볼드.

## 5. 금지

- 이모지 기능 아이콘(장식·배지용 이모지는 허용 — 배지/말머리 등 의미 전달 보조).
- 라이트 지도 타일 노출, 브라우저 기본 포커스 링, 앰버 대면적 배경(버튼 외),
- 새 폰트 추가(기존 Jakarta+Noto KR 스택 유지 — 로딩 성능).
