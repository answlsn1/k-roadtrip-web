# K-Riders 스팟 DB 운영 가이드 (Phase 0)

라이더 스팟(카페·전망포토·와인딩진입점·맛집·휴게쉼터·주유충전·정비·숙박)을 모으고 관리하는 방법. 목표는 완벽한 기록이 아니라 **속도** — 필수 6필드만 채우면 등록되고, 나머지는 나중에 보강한다.

절대 원칙: **속도·랩타임·랭킹 관련 값은 어떤 필드에도 적지 않는다.** `winding_grade`는 재미 등급이지 속도 지표가 아니다.

---

## 운영 방법 2가지

### 1. 현장 루트 — 라이딩 중 폰으로 즉시 등록

라이딩 하다가 스팟을 발견했을 때. 1분 안에 끝나는 걸 목표로 한다.

1. 폰으로 `k-roadtrip.app/motorcycle/admin/spots` 접속(북마크 추천)
2. 관리자 토큰 입력(처음 한 번만 — 이후 세션 유지)
3. 「📍 현재 위치 가져오기」로 좌표 자동 입력
4. 필수 6필드(이름·카테고리·좌표·지역·출처) + 사진 몇 장
5. 저장 → 폼이 바로 초기화되니 다음 스팟 계속 등록 가능

선택 필드(접근·도로 상태, 촬영 팁, 추천 시기·시간대, 체류 시간, 개인 평점 등)는 그때그때 여유 있을 때만 채운다. 등록 직후 상태는 항상 `draft`(비공개) — 나중에 목록 화면에서 확인·검증 후 `active`로 전환한다.

### 2. 데스크 루트 — 시트에서 대량 정리 후 한 번에 반영

여러 스팟을 미리 조사했거나, 현장에서 대충 적어둔 걸 정리할 때.

1. `data/spots-template.csv`를 열어 시트(엑셀/구글시트)에서 정리 — 28개 컬럼 그대로 사용, 자세한 규칙은 `data/spots-template-guide.md` 참고
2. CSV(UTF-8)로 내보내기
3. 검증: `npm run spots:validate -- 파일경로`
   - 틀린 값이 있으면 행 번호 + 이유를 알려준다(사람이 읽는 행 번호 — 시트의 그 행을 바로 고치면 됨)
   - 전부 통과할 때까지 반복
4. 반영(미리보기): `npm run spots:import -- 파일경로`
   - 실제로 저장하지 않고, "이건 새 스팟" / "이건 이미 있는 걸 고치는 것" / "이건 그대로라 건너뜀"으로 분류만 보여준다
5. 반영(실제 저장): 위 명령 뒤에 `--apply`를 붙여서 실행 → 화면에 "apply"라고 직접 타이핑해야 진행됨(오타 방지)

**같은 CSV를 나중에 또 돌려도 안전하다** — 이미 등록된 스팟은 "건너뜀" 또는 "갱신"으로 처리되고, 어드민 화면에서 이미 검증·공개해둔 스팟의 사진·상태는 CSV가 그 칸을 비워뒀다면 그대로 유지된다(덮어쓰지 않음).

#### 사전 준비(한 번만)

`spots:validate`(기존 DB와 대조할 때)와 `spots:import`는 Supabase에 직접 접속하므로, 실행 전에 터미널에서 환경변수를 설정해야 한다(`.env.local`은 자동으로 안 읽힌다):

```powershell
$env:NEXT_PUBLIC_SUPABASE_URL="..."
$env:SUPABASE_SERVICE_ROLE_KEY="..."
npm run spots:import -- data/spots-template.csv
```

값은 `.env.local` 파일에 이미 저장돼 있다.

---

## 두 방법을 같이 쓸 때

현장에서 폰으로 등록한 스팟도, 나중에 데스크 루트로 시트에 다시 옮겨 CSV로 재정리해도 괜찮다 — `slug`(이름+지역으로 자동 생성되는 고유 식별자) 기준으로 같은 스팟임을 알아보고 갱신하지, 중복으로 새로 만들지 않는다.

## 공개 전환

등록 직후 스팟은 전부 `draft`(비공개) 상태다. 어드민 목록 화면에서 확인 후 "공개로" 버튼을 눌러야 방문자(익명 사용자)에게 보인다 — 익명은 `active` 상태인 스팟만 볼 수 있다(DB 접근 권한 규칙, `supabase/migrations/0010_motorcycle_spots.sql`).
